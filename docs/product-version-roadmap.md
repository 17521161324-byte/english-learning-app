# 产品功能与版本迭代计划

## 1. 产品目标

这是一个中文界面、个人使用、离线优先的 Android 英语学习 App。

产品核心不是展示一组互相独立的练习入口，而是形成完整闭环：

```text
收集学习材料 -> 生成每日计划 -> 完成真实练习 -> 自动记录结果
-> 安排后续复习 -> 汇总学习统计 -> 调整下一阶段计划
```

界面延续 Notion 式清爽工具感。Chrome 插件和云同步放在移动端核心学习闭环稳定之后。

## 2. 产品与技术分工

- 产品经理负责：场景、优先级、练习规则、页面文案和验收体验。
- 技术经理负责：技术选型、数据模型、版本边界、风险控制、测试和发布流程。
- 每个版本开发前冻结范围；版本内只处理阻断问题，不临时加入大功能。
- 每个正式版本必须能够通过现有 App 更新入口安装；原生版发布后的数据升级必须保持兼容。

## 3. 技术路线

### 3.1 原生重建

下一代客户端采用 Kotlin + Jetpack Compose，从空数据开始重建：

- 当前 Capacitor 版本只作为需求和视觉参考，不再继续迭代。
- 不迁移 IndexedDB 数据，也不要求原生版覆盖旧版全部功能。
- 原生测试包使用独立的开发包名，可以和当前 App 同时安装。
- 正式发布时沿用当前 applicationId 和 release keystore，继续支持应用内更新。
- 首个原生正式版本从空数据库开始；从该版本起，后续 Room 数据升级必须兼容。

### 3.2 Android 技术选型

- UI：Jetpack Compose + Material 3，自定义为现有 Notion 工具风格。
- 架构：单 Activity、Navigation Compose、按功能分包的 MVVM。
- 本地数据库：Room，所有结构升级都提供 Migration。
- 轻量配置：DataStore。
- 后台任务：WorkManager，用于复习提醒、备份和内容下载。
- 录音：Android MediaRecorder；播放使用 Media3。
- 网络：OkHttp + Retrofit，网络内容下载后可离线使用。
- 依赖管理：Alpha 使用轻量 AppContainer；功能模块增加后再引入 Hilt，避免初期构建复杂度。
- 更新：GitHub Release、latest.json、DownloadManager 和系统安装器。
- 复习算法：采用经过验证的 FSRS 实现，不自行设计记忆算法。
- 后端：核心版本不需要；Chrome 跨设备同步阶段再接入 Supabase。

### 3.3 核心数据对象

- Word：单词及基础信息。
- MeaningSense：词义、词性和语境，解决一词多义问题。
- Example：例句、原文上下文和来源网址。
- ReviewState：熟练度、到期时间和复习参数。
- StudyEvent：每次学习行为、结果、用时和来源。
- StudyPlan：用户设置的每日目标。
- PlanProgress：由 StudyEvent 自动计算的计划进度。
- WordRelation：近义、反义、易混、词族和搭配关系。
- Phrase：短语、释义、例句和掌握状态。
- SpeechContent / SpeechSegment：演讲及其章节、句子。
- Recording：录音文件路径、时长、练习类型和关联内容。

StudyEvent 是自动计划和统计的统一依据。练习功能不得各自维护互不一致的计数。

## 4. 版本路线

### v0.2.0-alpha：原生基础工程

目标：建立可持续开发和发布的原生底座，不进入正式更新通道。

范围：

- Compose 应用框架、导航、主题和通用组件。
- Room 数据库、DataStore 和 StudyEvent 事件模型。
- GitHub Actions 签名构建、测试 APK 和崩溃日志导出。
- 应用内检查更新、下载 APK、调起系统安装器。
- 原生数据导入导出协议和空数据库初始化。
- 开发包与正式包并行安装。

退出条件：测试 APK 可安装；重启后数据不丢失；更新检查正常；空数据库初始化稳定。

### v0.3.0-beta：原生核心学习版

目标：建立新原生版本的首个完整学习闭环，不受旧版功能范围限制。

范围：

- 首页、生词库、单词详情、新增、编辑、删除、搜索和筛选。
- 例句、网页来源、个人笔记和发音播放。
- 基础复习与 FSRS 排期。
- 场景、跟读、自由表达三种口语入口。
- 原生录音、播放和删除。
- 原生 JSON 备份和恢复。

退出条件：从新增单词到复习和口语记录的核心流程完整；录音权限与生命周期稳定；备份恢复后数据一致。

### v0.4.0：拼词训练

目标：形成从“认识单词”到“主动拼写”的学习闭环。

范围：

- 中文释义和词性提示下完整输入英文。
- 词根、前后缀或音节优先的乱序词块组合。
- 部分隐藏和全部隐藏两种辅助模式。
- 首字母、揭示字母和发音提示。
- 大小写、连字符、撇号和多义词答案规则。
- 错误位置反馈、再次尝试和结果记录。

退出条件：四种拼词模式均写入 StudyEvent；退出练习后可恢复进度；正确率和提示次数可统计。

### v0.5.0：自动计划与学习统计

目标：首页计划完全由真实学习行为驱动。

范围：

- 设置每日新词、复习、拼词、短语、口语、演讲和总时长目标。
- 根据到期复习项和用户目标生成当天任务。
- 点击任务进入带有正确筛选条件的对应练习。
- 按有效次数、正确率或时长自动更新完成状态。
- 删除录音或学习记录后自动重新计算进度。
- 总计、7 天、30 天、本月和自定义时间统计。
- 学习天数、时长、词汇、复习、拼词正确率、口语和计划完成率。

退出条件：首页不存在手动勾选的学习任务；计划进度和统计均可从 StudyEvent 重建并得到相同结果。

### v0.6.0：关联词与短语学习

目标：从孤立记忆升级到词义网络和表达组合。

范围：

- 按具体词义展示近义词、反义词、易混词、词族和常见搭配。
- 中文概念对应多个英文表达的语境辨析。
- 网络候选必须经用户确认后进入个人词库。
- 短语新增、收藏、编辑、删除和来源记录。
- 短语完整输入、挖空、乱序重组和听写。
- 单词详情与短语、关联词双向跳转。

退出条件：所有关系绑定 MeaningSense 而不是只绑定单词；离线时已保存关系和短语可正常学习。

### v0.7.0：演讲观看与背诵

目标：支持长内容的理解、跟读和分段背诵。

范围：

- 公共领域演讲内容库和用户自行导入。
- 演讲详情、章节、逐句文本、音视频和学习进度。
- 逐句理解、跟读、影子跟读、关键词提示和分段背诵。
- 句子录音、回听、删除和有效时长检查。
- 按句数或学习分钟数接入每日计划。
- 内容下载和离线管理。

退出条件：版权来源和授权信息可追溯；长内容可以断点续学；无网络时已下载内容可播放和学习。

### v0.8.0：Chrome 插件与同步

目标：打通电脑网页阅读和手机学习。

范围：

- 浏览器划词收藏、释义选择、原句和来源网址保存。
- Supabase 个人账号、设备同步和冲突处理。
- 手机接收网页生词并自动归入待学习队列。
- 同步状态、失败重试和手动刷新。
- 云端加密传输、数据导出和账号删除。

退出条件：离线收藏可排队；重复单词不会丢失不同网页语境；手机和插件冲突有确定合并规则。

### v1.0.0：稳定个人版

目标：形成可长期使用、更新和恢复的数据资产。

范围：

- 全量备份、恢复和数据完整性检查。
- 数据库升级、异常恢复、性能和耗电优化。
- 核心流程自动化测试和 Android 真机回归。
- 权限、隐私、版权说明和错误反馈完善。
- 正式版本发布与回滚手册。

退出条件：连续使用和跨版本升级不丢数据；关键流程无阻断缺陷；签名和更新链路完成灾难恢复演练。

## 5. 暂缓范围

以下能力在 v1.0 前不承诺：

- 静默安装 APK，普通 Android 应用无法实现。
- 自动判断用户发音是否标准；没有可靠评分服务前只记录和回听。
- 未确认的 AI 关联词直接加入复习。
- 抓取或分发没有明确授权的完整演讲稿和视频。
- 社区、排行榜、好友和付费体系。
- iOS 客户端。

## 6. 版本工作流

每个版本按以下步骤推进：

1. 产品场景确认：明确入口、步骤、空状态、异常状态和完成规则。
2. 原型确认：完整走通主要交互，不以静态页面代替行为验证。
3. 技术设计：确定数据变更、权限、网络条件和迁移方案。
4. 开发与测试：单元测试、数据库迁移测试和真机回归。
5. Beta 验证：安装测试包，由产品经理按验收清单体验。
6. 正式发布：提高 versionCode，生成签名 APK、Release 和 latest.json。
7. 发布复盘：记录问题和真实使用反馈，再决定下一版本范围。

## 7. 当前下一步

当前进入 v0.2.0-alpha 需求冻结。开发前需要完成以下产品确认：

- 首次启动、空数据引导和基础资料初始化流程。
- 首页、词库、复习、口语和设置的原生信息架构。
- 正式版与开发测试版的名称和图标区分。
- 权限申请、无网络、存储失败和录音失败的处理流程。
